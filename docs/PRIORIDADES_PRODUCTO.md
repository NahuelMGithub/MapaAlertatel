# Prioridades de Producto - Alertatel CRM Comercial

## Decisión de alcance

No vamos a construir todo el CRM de golpe.

La dirección actual es construir una herramienta comercial enfocada en municipios, con dos modos principales de uso y un conjunto reducido de funcionalidades que produzcan valor rápido para el equipo comercial.

El producto debe crecer desde una base simple, clara y accionable. Cada nueva función debe ayudar a decidir mejor, contactar mejor o hacer seguimiento con más orden.

## Funcionalidades prioritarias

Las funcionalidades prioritarias son:

1. Estado comercial claro por cuenta.
2. Próxima acción obligatoria.
3. Contactos por municipio.
4. Productos contratados.
5. Señales comerciales automáticas.

Estas cinco capacidades definen el primer producto útil. Todo lo demás debe considerarse soporte, futuro o experimental hasta que estas piezas funcionen bien.

## 1. Estado comercial claro por cuenta

Cada municipio debe tener un estado comercial simple y visible.

Estados iniciales sugeridos:

- Cliente.
- Prospecto.
- Contacto iniciado.
- Propuesta enviada.
- Negociación.
- Descartado.
- Sin datos.

Valor para el usuario:

- Entender rápidamente dónde está cada municipio.
- Evitar confusión entre cliente, prospecto y oportunidad real.
- Priorizar acciones sin revisar notas dispersas.

## 2. Próxima acción obligatoria

Cada cuenta que esté activa comercialmente debe tener una próxima acción.

Ejemplos:

- Llamar.
- Pedir contacto.
- Enviar propuesta.
- Coordinar reunión.
- Hacer seguimiento.
- Actualizar datos.
- Revisar señal comercial.

Valor para el usuario:

- Evitar municipios abandonados.
- Convertir información en acción.
- Ordenar el trabajo diario y semanal.

## 3. Contactos por municipio

El sistema debe permitir identificar personas relevantes de cada municipio.

Información prioritaria:

- Nombre.
- Cargo o área.
- Teléfono.
- Email, si existe.
- Rol comercial: decisor, técnico, administrativo, referente o desconocido.
- Observaciones.
- Fecha de último contacto.

Valor para el usuario:

- Saber con quién hablar.
- Reducir dependencia de memoria individual.
- Mejorar preparación de reuniones y llamadas.

## 4. Productos contratados

Para clientes, el sistema debe mostrar qué tiene contratado cada municipio.

Información prioritaria:

- Producto.
- Cantidad.
- Monto.
- Fecha de inicio.
- Estado del contrato o servicio.
- Observaciones comerciales.

Valor para el usuario:

- Cuidar clientes actuales.
- Detectar ampliaciones posibles.
- Entender valor actual de la cuenta.

## 5. Señales comerciales automáticas

Las señales comerciales son alertas o indicios que ayudan a priorizar.

Tipos prioritarios:

- Noticias relevantes.
- Cambios institucionales.
- Falta de seguimiento.
- Patrones de datos.
- Oportunidades detectadas automáticamente.

Valor para el usuario:

- Saber qué municipios requieren atención.
- Detectar oportunidades sin depender solo de revisión manual.
- Convertir datos públicos e internos en acciones comerciales.

## Modos principales del sistema

El sistema debe organizarse en dos modos de trabajo.

## A. Modo exploración

Sirve para investigar, comparar y preparar acciones comerciales.

Incluye:

- Mapa.
- Búsqueda de municipios.
- Análisis de prospectos.
- Comparación territorial.
- Preparación de reuniones.

Preguntas que responde:

- ¿Dónde hay clientes y prospectos?
- ¿Qué municipios tienen más potencial?
- ¿Qué información necesito antes de llamar?
- ¿Cómo se compara este municipio con otros?
- ¿Qué señales sugieren oportunidad?

## B. Modo ejecución

Sirve para trabajar, avanzar y dar seguimiento.

Incluye:

- Trabajo diario con checklist.
- Trabajo semanal.
- Trabajo mensual estilo OKR.
- Seguimiento de tareas.
- Seguimiento de avances.
- Seguimiento de prioridades.

Preguntas que responde:

- ¿Qué tengo que hacer hoy?
- ¿Qué está vencido?
- ¿Qué cuentas están avanzando?
- ¿Qué objetivos comerciales del mes están pendientes?
- ¿Qué señales tengo que revisar?

## Ideas futuras o fuera del foco inicial

Estas ideas pueden ser útiles, pero no son prioridad inmediata:

- Pipeline comercial completo.
- Gestión avanzada de documentos.
- Calendario integrado.
- Reuniones con minutas avanzadas.
- Permisos complejos por usuario.
- Dashboard ejecutivo avanzado.
- Automatización completa de campañas.
- Integraciones externas múltiples.
- IA generativa para mensajes comerciales.

No están descartadas. Simplemente no deben competir con las cinco funcionalidades prioritarias.

## Ideas descartadas por ahora

Por ahora no conviene:

- Construir un CRM genérico.
- Copiar un pipeline tipo HubSpot completo.
- Diseñar un sistema empresarial de permisos desde el inicio.
- Automatizar decisiones comerciales sin validación humana.
- Usar IA antes de tener estado, contactos, productos, acciones y señales bien ordenadas.

## Próximo paso recomendado

Antes de avanzar con base de datos o IA, el sistema debería permitir editar y visualizar correctamente:

1. Estado comercial.
2. Próxima acción.
3. Contactos.
4. Productos contratados.
5. Señales comerciales iniciales, aunque empiecen como placeholders o reglas simples.

Una vez que esto funcione, se puede fortalecer el modo ejecución con trabajo diario, semanal y mensual.
